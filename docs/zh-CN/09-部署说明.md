## 1. 环境要求

### 1.1 后端环境

| 软件 | 最低版本 | 推荐版本 | 说明 |
|---|---|---|---|
| JDK | 17 | 17 LTS | 编译和运行 Java 代码 |
| Maven | 3.9 | 3.9+ | 项目依赖管理和构建 |

> **注意**：项目包含 Maven Wrapper (`mvnw`)，在没有全局安装 Maven 的环境下也可使用 `./mvnw` 命令启动。

### 1.2 前端环境

| 软件 | 最低版本 | 推荐版本 | 说明 |
|---|---|---|---|
| Node.js | 18 | 18 LTS / 20 LTS | JavaScript 运行时 |
| npm | 9 | 9+ | 随 Node.js 18+ 附带 |
| pnpm (可选) | 8 | 8+ | 更快的包管理器 |

### 1.3 验证环境

使用以下命令验证环境是否就绪：

```bash
# 验证 JDK
java -version
# 期望输出: openjdk version "17.x.x" ...

# 验证 Maven
mvn -version
# 期望输出: Apache Maven 3.9.x ...

# 验证 Node.js
node -v
# 期望输出: v18.x.x 或 v20.x.x

# 验证 npm
npm -v
# 期望输出: 9.x.x 或 10.x.x
```

## 3. 后端部署与启动

### 3.1 使用 Maven 启动 (开发模式)

```bash
# 进入后端目录
cd foli-backend

# 使用 Maven Wrapper 编译并启动
# Windows:
mvnw.cmd spring-boot:run

# Linux / Mac:
./mvnw spring-boot:run
```

### 3.2 使用 IDE 启动

**IntelliJ IDEA**：

1. File -> Open -> 选择 `foli-backend` 目录
2. 等待 Maven 依赖同步完成
3. 找到 `FoliBackendApplication.java`
4. 右键 -> Run 'FoliBackendApplication'

**VS Code**：

1. 安装 Spring Boot Extension Pack 插件
2. 打开 `foli-backend` 目录
3. 在 Spring Boot Dashboard 中启动

### 3.3 编译为 JAR 包运行

```bash
cd foli-backend

# 编译打包（跳过测试）
./mvnw clean package -DskipTests

# 运行 JAR
java -jar target/foli-backend-0.0.1-SNAPSHOT.jar
```

### 3.4 启动成功标志

控制台输出类似以下日志表示启动成功：

```
Started FoliBackendApplication in X.XXX seconds
...
Tomcat started on port 8080 (http)
```

## 4. 前端部署与启动

### 4.1 安装依赖

```bash
# 进入前端目录
cd foli-frontend

# 安装依赖
npm install
# 或使用 pnpm:
pnpm install
```

### 4.2 开发模式启动

```bash
npm run dev
```

启动成功后：

```
VITE vX.X.X  ready in XXX ms

-> Local:   http://localhost:5173/
```

### 4.3 生产构建

```bash
npm run build
```

构建产物输出到 `foli-frontend/dist/` 目录，可将该目录部署到 Nginx 或其他静态文件服务器。

#### Nginx 配置示例

```nginx
server {
    listen       80;
    server_name  localhost;

    # 前端静态资源
    location / {
        root   /path/to/foli-frontend/dist;
        index  index.html;
        try_files $uri $uri/ /index.html;  # Vue History 模式
    }

    # 代理 API 请求到后端
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 代理上传文件
    location /uploads/ {
        proxy_pass http://localhost:8080;
    }
}
```

## 5. 访问地址汇总

启动完成后，可通过以下地址访问系统各组件：

| 组件 | 地址 | 说明 |
|---|---|---|
| 后端 API 服务 | `http://localhost:8080` | REST API 基础地址 |
| Knife4j API 文档 | `http://localhost:8080/doc.html` | 交互式 API 文档，支持在线调试 |
| Swagger UI | `http://localhost:8080/swagger-ui.html` | 标准 Swagger UI 界面 |
| OpenAPI 3.0 JSON | `http://localhost:8080/v3/api-docs` | OpenAPI 规范 JSON |
| H2 数据库控制台 | `http://localhost:8080/h2-console` | H2 Web 管理界面 |
| 前端应用 | `http://localhost:5173` | Vue 3 开发服务器 |

### 5.1 H2 控制台连接信息

| 参数 | 值 |
|---|---|
| JDBC URL | `jdbc:h2:mem:foli_mall;MODE=MySQL;DB_CLOSE_DELAY=-1;DATABASE_TO_UPPER=false` |
| 用户名 | `sa` |
| 密码 | (空) |

### 5.2 Knife4j 文档使用

1. 打开 `http://localhost:8080/doc.html`
2. 在页面右上角可以切换文档分组和语言
3. 需要认证的接口：先在 "认证管理 Auth" 模块执行登录，获取 Token
4. 点击页面右上角的 "Authorize" 按钮，输入 `Bearer <token>` 进行全局认证
5. 认证后可直接在页面上调试所有需要登录的接口

## 6. 配置说明

### 6.1 核心配置文件

配置文件位于 `foli-backend/src/main/resources/application.yaml`。

### 6.2 数据库配置

```yaml
spring:
  datasource:
    url: jdbc:h2:mem:foli_mall;MODE=MySQL;DB_CLOSE_DELAY=-1;DATABASE_TO_UPPER=false
    driver-class-name: org.h2.Driver
    username: sa
    password:
```

| 参数 | 说明 |
|---|---|
| `MODE=MySQL` | H2 以 MySQL 兼容模式运行 |
| `DB_CLOSE_DELAY=-1` | JVM 存活期间保持数据库不关闭 |
| `DATABASE_TO_UPPER=false` | 表名/列名不强制转为大写 |

**切换为文件持久化模式**（需要数据持久化时）：

```yaml
spring:
  datasource:
    url: jdbc:h2:file:./data/foli_mall;MODE=MySQL;DATABASE_TO_UPPER=false
```

### 6.3 JWT 配置

```yaml
jwt:
  secret: foli-mall-secret-key-2026-please-change-in-production-environment
  expiration: 86400000   # 24小时 = 24 * 60 * 60 * 1000 毫秒
```

| 参数 | 说明 |
|---|---|
| `jwt.secret` | JWT 签名密钥，生产环境务必更换为强随机字符串 |
| `jwt.expiration` | Token 有效期，单位毫秒。默认 86,400,000ms = 24 小时 |

### 6.4 文件上传配置

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB

app:
  upload:
    path: ./uploads   # 上传文件存储目录
```

| 参数 | 说明 |
|---|---|
| `max-file-size` | 单个上传文件的最大大小 |
| `max-request-size` | 单次请求的最大大小 |
| `app.upload.path` | 上传文件在服务器上的存储路径，相对路径相对于项目根目录 |

### 6.5 服务端口

```yaml
server:
  port: 8080
```

默认端口 8080，如有冲突可修改。

### 6.6 MyBatis-Plus 配置

```yaml
mybatis-plus:
  global-config:
    db-config:
      id-type: ASSIGN_ID         # 雪花 ID 策略
      logic-delete-field: isDelete
      logic-delete-value: 1
      logic-not-delete-value: 0
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl  # 控制台打印 SQL
    map-underscore-to-camel-case: true  # 下划线转驼峰
```

## 7. 常见问题排查

### 7.1 端口被占用

**错误信息**：`Port 8080 was already in use`

**解决方案**：

```bash
# 查找占用端口的进程
# Windows:
netstat -ano | findstr :8080

# Linux / Mac:
lsof -i :8080

# 修改端口：在 application.yaml 中修改 server.port
```

### 7.2 JDK 版本不匹配

**错误信息**：`java.lang.UnsupportedClassVersionError`

**解决方案**：确保使用 JDK 17 或更高版本。

```bash
# 检查当前版本
java -version

# 如果安装了多个 JDK 版本，设置 JAVA_HOME
# Windows:
set JAVA_HOME=C:\Program Files\Java\jdk-17
# Linux / Mac:
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
```

### 7.3 Maven 依赖下载失败

**解决方案**：

```bash
# 清除本地仓库缓存后重新下载
rm -rf ~/.m2/repository/com/github/foli-backend
./mvnw clean install -U
```

### 7.4 前端 npm install 失败

**解决方案**：

```bash
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 和 lock 文件后重新安装
rm -rf node_modules package-lock.json
npm install
```

### 7.5 前端代理配置

如果前端开发时需要代理 API 请求到后端，检查 `foli-frontend/vite.config.ts` 中的代理配置：

```typescript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
})
```

### 7.6 H2 控制台无法连接

确保 JDBC URL 与配置文件中的完全一致，特别是以下参数：

- `DB_CLOSE_DELAY=-1`（必须）
- `DATABASE_TO_UPPER=false`（必须，否则表名大写导致找不到表）

### 7.7 Knife4j 文档无法访问

确保 `application.yaml` 中 Knife4j 已启用：

```yaml
knife4j:
  enable: true
```

访问路径为 `/doc.html`，注意不是 `/swagger-ui.html`（后者是标准 Swagger UI）。
