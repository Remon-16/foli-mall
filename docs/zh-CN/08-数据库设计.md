## 2. ER 关系说明

### 2.1 核心实体关系图

```
fm_user (用户)
  | 1
  | (user_id)
  |
  +--< fm_store (店铺)          [1:N] 一个用户(SELLER)可有多个店铺
  |     | 1
  |     | (store_id)
  |     |
  |     +--< fm_product (商品)  [1:N] 一个店铺有多个商品
  |           | 1
  |           | (product_id)
  |           |
  |           +--< fm_product_image (商品图片) [1:N] 一个商品有多张图片
  |
  +--< fm_cart_item (购物车)    [1:N] 一个用户(BUYER)有多个购物车项
  |     |
  |     | (product_id)
  |     v
  |     fm_product (商品)
  |
  +--< fm_order (订单)          [1:N] 一个用户(BUYER)有多个订单
  |     | 1
  |     | (order_id)
  |     |
  |     +--< fm_order_item (订单明细) [1:N] 一个订单有多个明细
  |
  +--< fm_return_refund (退货)  [1:N] 一个用户(BUYER)可有多个退货申请
  |
  +--< fm_complaint (投诉)      [1:N] 一个用户(BUYER)可发起多个投诉
  |
  +--< fm_message (消息)        [1:N] 一个用户可发送/接收多条消息
  |
  +--< fm_balance_log (余额流水) [1:N] 一个用户有多条余额流水

fm_product_category (分类)
  | 1
  | (parent_id 自引用)
  |
  +--< fm_product_category  [树形自关联]
  |
  +--< fm_product (商品)   [1:N] 一个分类下有多个商品
```

### 2.2 关键外键关系

| 从表 | 外键字段 | 引用表 | 引用字段 | 关系说明 |
|---|---|---|---|---|
| `fm_store` | `user_id` | `fm_user` | `id` | 店铺 -> 店主 |
| `fm_product` | `store_id` | `fm_store` | `id` | 商品 -> 店铺 |
| `fm_product` | `category_id` | `fm_product_category` | `id` | 商品 -> 分类 |
| `fm_product_image` | `product_id` | `fm_product` | `id` | 图片 -> 商品 |
| `fm_cart_item` | `user_id` | `fm_user` | `id` | 购物车 -> 买家 |
| `fm_cart_item` | `product_id` | `fm_product` | `id` | 购物车 -> 商品 |
| `fm_order` | `user_id` | `fm_user` | `id` | 订单 -> 买家 |
| `fm_order` | `store_id` | `fm_store` | `id` | 订单 -> 店铺 |
| `fm_order_item` | `order_id` | `fm_order` | `id` | 订单明细 -> 订单 |
| `fm_message` | `sender_id` | `fm_user` | `id` | 消息 -> 发送者 |
| `fm_message` | `receiver_id` | `fm_user` | `id` | 消息 -> 接收者 |
| `fm_complaint` | `user_id` | `fm_user` | `id` | 投诉 -> 投诉人 |
| `fm_complaint` | `store_id` | `fm_store` | `id` | 投诉 -> 被投诉店铺 |
| `fm_complaint` | `order_id` | `fm_order` | `id` | 投诉 -> 关联订单（可选） |
| `fm_complaint` | `return_id` | `fm_return_refund` | `id` | 投诉 -> 关联退货（可选） |
| `fm_return_refund` | `order_id` | `fm_order` | `id` | 退货 -> 关联订单 |
| `fm_return_refund` | `user_id` | `fm_user` | `id` | 退货 -> 买家 |
| `fm_return_refund` | `store_id` | `fm_store` | `id` | 退货 -> 店铺 |
| `fm_balance_log` | `user_id` | `fm_user` | `id` | 余额流水 -> 用户 |

> **注意**：H2 内存数据库中以上外键关系是逻辑上的引用关系，DDL 中未显式创建外键约束（FOREIGN KEY），以减少维护成本和提升灵活性。

## 4. 通用字段规范

### 4.1 标准字段

所有表统一包含以下通用字段：

| 字段名 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT | 主键，雪花 ID 算法生成 |
| `createTime` | TIMESTAMP | 创建时间，默认 CURRENT_TIMESTAMP |
| `editTime` | TIMESTAMP | 编辑时间，默认 CURRENT_TIMESTAMP（MyBatis-Plus 自动填充策略可覆盖） |
| `updateTime` | TIMESTAMP | 更新时间，默认 CURRENT_TIMESTAMP（MyBatis-Plus 自动填充） |
| `isDelete` | TINYINT | 逻辑删除标记，0=正常, 1=已删除 |

### 4.2 主键策略

- **算法**: Twitter Snowflake（雪花算法）
- **配置**: MyBatis-Plus `id-type: ASSIGN_ID`
- **特点**: 全局唯一、趋势递增、不含业务含义
- **实现**: 使用 `cn.hutool.core.util.IdUtil` 或 MyBatis-Plus 内置的 `IdentifierGenerator`

### 4.3 逻辑删除

- **字段**: `isDelete`
- **配置**: MyBatis-Plus `logic-delete-field: isDelete`
- **删除值**: 1
- **未删除值**: 0
- **效果**: 所有查询自动附加 `WHERE isDelete = 0`；删除操作自动转为 `UPDATE SET isDelete = 1`

### 4.4 时间字段

| 字段 | 说明 | 填充策略 |
|---|---|---|
| `createTime` | 记录创建时间，插入时设置 | 数据库默认值 `CURRENT_TIMESTAMP` |
| `editTime` | 记录编辑时间 | 数据库默认值 `CURRENT_TIMESTAMP`，可由 MyBatis-Plus 填充策略覆盖 |
| `updateTime` | 记录更新时间，每次更新自动刷新 | MyBatis-Plus 自动填充 `FieldFill.INSERT_UPDATE` |

## 6. 初始化数据策略

### 6.1 表结构初始化

- 通过 Spring Boot 的 `spring.sql.init.mode: always` 自动执行 `classpath:schema.sql`
- `schema.sql` 包含完整的 12 张表的 `CREATE TABLE IF NOT EXISTS` 语句
- 使用 `IF NOT EXISTS` 保证幂等性

### 6.2 预置数据

- 预置数据通过 `sql/data.sql` 文件提供（独立于 classpath）
- 预置内容包括：
  - 管理员账号
  - 卖家账号 + 店铺
  - 商品分类（树形结构）
  - 示例商品
  - 示例订单（覆盖多种状态）
- 数据量适中，既保证覆盖各种测试场景，又不影响启动速度

### 6.3 数据重置

- 重启应用即可重置所有数据
- H2 内存数据库模式：进程结束即清空，重启时从 SQL 文件重新初始化
- H2 文件模式（生产环境不建议）：需手动删除数据库文件
